# 🔑 Двери и ключи - Полная инструкция

## 📋 Что изменилось

**Главное улучшение:** Теперь система использует **РАЗДЕЛЬНЫЕ UI панели** для подсказок и сообщений, что устраняет конфликты!

---

## 🎨 Структура UI

Вам нужно создать **4 UI панели** (все под Canvas):

### 1️⃣ Door Prompt Panel (подсказка двери)
```
Canvas → Panel → Text: "[E] Открыть"
- Назовите: "DoorPromptPanel"
- Скройте по умолчанию (галочка Panel ❌)
- Настройте внешний вид (фон, текст, позиция)
```

### 2️⃣ Door No Key Message Panel (сообщение "нет ключа")
```
Canvas → Panel → Text: "Требуется ключ..."
- Назовите: "DoorNoKeyMessagePanel"
- Скройте по умолчанию (галочка Panel ❌)
- Эта панель показывается ТОЛЬКО когда нет ключа
```

### 3️⃣ Key Prompt Panel (подсказка ключа)
```
Canvas → Panel → Text: "[E] Подобрать"
- Назовите: "KeyPromptPanel"
- Скройте по умолчанию (галочка Panel ❌)
- Показывается когда игрок рядом с ключом
```

### 4️⃣ Key Pickup Message Panel (сообщение "ключ подобран")
```
Canvas → Panel → Text: "🔑 Ключ подобран!"
- Назовите: "KeyPickupMessagePanel"
- Скройте по умолчанию (галочка Panel ❌)
- Показывается после подбора ключа
```

## 🛠 Настройка в Unity

### 1️⃣ GameManager (если ещё нет)

```
1. GameObject → Create Empty → "GameManager"
2. Add Component → GameManager
3. В инспекторе:
   - Player: перетащи игрока (или убедись что есть тег "Player")
   - Pause Menu: ваша панель паузы (если есть)
```

### 2️⃣ Ключ (KeyPickup)

```
1. Создай 3D объект ключа
2. Add Component → KeyPickup
3. Настрой в инспекторе:

   [Настройки ключа]
   - Key Name: "UpperHallKey" (уникальное имя)
   - Pickup Radius: 2
   - Destroy After Pickup: ✅

   [UI - Подсказки и сообщения]
   - Prompt Panel: перетащи "KeyPromptPanel"
   - Pickup Message Panel: перетащи "KeyPickupMessagePanel"
   - Pickup Message Text: перетащи Text внутри KeyPickupMessagePanel
   - Message Duration: 2

4. Важно: Убедись что Prompt Panel и Message Panel - это РАЗНЫЕ объекты!
```

### 3️⃣ Дверь БЕЗ ключа

```
1. Создай 3D объект двери
2. Add Component → Door
3. Настрой в инспекторе:

   [Настройки двери]
   - Interaction Radius: 2
   - Requires Key: ❌ (выключи)

   [Анимация]
   - Rotation Axis: (0, 1, 0)
   - Open Angle: 90
   - Open Duration: 1.5

   [UI - Подсказки и сообщения]
   - Prompt Panel: перетащи "DoorPromptPanel"
   - No Key Message Panel: оставь пустым (не нужен)
   - No Key Message Text: оставь пустым

4. Поставь дверь в закрытое положение перед запуском!
```

### 4️⃣ Дверь С ключом

```
1. Создай 3D объект двери
2. Add Component → Door
3. Настрой в инспекторе:

   [Настройки двери]
   - Interaction Radius: 2
   - Requires Key: ✅ (включи)
   - Required Key Name: "UpperHallKey" (должно совпадать с KeyPickup!)

   [Анимация]
   - Rotation Axis: (0, 1, 0)
   - Open Angle: 90
   - Open Duration: 1.5

   [UI - Подсказки и сообщения]
   - Prompt Panel: перетащи "DoorPromptPanel"
   - No Key Message Panel: перетащи "DoorNoKeyMessagePanel"
   - No Key Message Text: перетащи Text внутри DoorNoKeyMessagePanel
   - Message Duration: 2

4. Важно: Prompt Panel и No Key Message Panel - РАЗНЫЕ объекты!
```

## 🎮 Как теперь работает

### Ключ:
| Событие | Что происходит |
|---------|----------------|
| **Подойти к ключу** | Появляется "[E] Подобрать" (KeyPromptPanel) |
| **Нажать E** | Ключ подобран, появляется "🔑 Подобран ключ" (KeyPickupMessagePanel) на 2 сек |
| **Ключ исчезает** | Если Destroy After Pickup = ✅ |
| **Отойти от ключа** | "[E] Подобрать" исчезает |

### Дверь без ключа:
| Событие | Что происходит |
|---------|----------------|
| **Подойти** | Появляется "[E] Открыть" (DoorPromptPanel) |
| **Нажать E** | Дверь плавно открывается (1.5 сек) |
| **После открытия** | Больше нельзя взаимодействовать |

### Дверь с ключом (НЕТ ключа):
| Событие | Что происходит |
|---------|----------------|
| **Подойти** | Появляется "[E] Открыть" (DoorPromptPanel) |
| **Нажать E** | Появляется "Требуется ключ: UpperHallKey" (DoorNoKeyMessagePanel) на 2 сек |
| **Подсказка "[E] Открыть"** | НЕ исчезает, можно попробовать снова |

### Дверь с ключом (ЕСТЬ ключ):
| Событие | Что происходит |
|---------|----------------|
| **Подойти** | Появляется "[E] Открыть" (DoorPromptPanel) |
| **Нажать E** | Дверь открывается, ключ НЕ расходуется |

## ⚠️ Важные моменты

### ✅ Правильная структура:
```
Каждый объект (Дверь/Ключ) использует:
- СВОЮ подсказку (Prompt Panel) - показывается при приближении
- СВОЁ сообщение (Message Panel) - показывается после действия

Эти панели НЕ пересекаются и НЕ конфликтуют!
```

### ❌ Чего больше НЕ будет:
- ~~Подсказка не исчезает при показе сообщения~~
- ~~Сообщение не перекрывает подсказку~~
- ~~Окно подбора ключа не застревает~~
- ~~Конфликты между UI двери и ключа~~

## 📝 Пример сцены

```
Сцена:
├─ GameManager
│   └─ Player: [Player объект]
├─ Canvas
│   ├─ DoorPromptPanel (скрыта)
│   ├─ DoorNoKeyMessagePanel (скрыта)
│   ├─ KeyPromptPanel (скрыта)
│   └─ KeyPickupMessagePanel (скрыта)
├─ Key (KeyPickup)
│   ├─ Key Name: "UpperHallKey"
│   ├─ Prompt Panel: KeyPromptPanel
│   └─ Pickup Message Panel: KeyPickupMessagePanel
└─ Door (Door)
    ├─ Requires Key: ✅
    ├─ Required Key Name: "UpperHallKey"
    ├─ Prompt Panel: DoorPromptPanel
    └─ No Key Message Panel: DoorNoKeyMessagePanel
```

**Порядок действий в игре:**
1. Подойти к Key → "[E] Подобрать" → нажать E → "🔑 Подобран ключ"
2. Подойти к Door → "[E] Открыть" → нажать E → дверь открывается

## 🐛 Если что-то не работает

**Сообщение не появляется:**
```
- Проверь что Message Panel назначен в инспекторе
- Проверь что Message Text назначен в инспекторе
- Смотри консоль Unity на ошибки
```

**Подсказка не исчезает:**
```
- Убедись что Prompt Panel и Message Panel - разные объекты!
- Проверь что Prompt Panel скрыт по умолчанию
```

**Дверь не открывается:**
```
- Проверь Requires Key и Required Key Name
- Убедись что ключ подобран (смотри консоль)
- Проверь что дверь ещё не открыта
```
